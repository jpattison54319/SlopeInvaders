# Agent Principles for Slope Invaders

## 1. Do not build a worksheet with game decorations

Slope Invaders should be a true game-based learning experience. The game and learning mechanics must be integrated.

**Good feature:** A shield blocks horizontal shots, forcing students to adjust slope and intercept.

**Weak feature:** A math question appears after an unrelated arcade sequence.

### Agent rule

Before adding any feature, answer:

> What mathematical behavior does this feature make the student practice?

If the answer is unclear, do not add the feature yet.

## 2. Preserve the game core

The core loop is:

1. Read the coordinate grid and target constraints.
2. Choose or adjust a linear equation.
3. Fire the laser along the graphed line.
4. Receive visual and mathematical feedback.
5. Revise strategy or progress.

Any new mechanic should strengthen this loop, not replace it.

## 3. Introduce one new burden at a time

Do not introduce slope, intercept, quadrants, shields, linked targets, friendly ships, and no-preview mode all at once.

Use this rhythm:

1. Introduce
2. Practice
3. Vary
4. Challenge
5. Mastery check
6. Reflect

## 4. Feedback must teach, not merely judge

Bad feedback:

> Missed.

Good feedback:

> Miss. At x = 4, your equation gave y = 2, but the asteroid was at y = 6. Your shot was 4 units too low.

## 5. Use supports, then fade them

Early levels should include strong scaffolds such as trajectory preview, visible coordinates, and slope triangles. Later levels should reduce those supports as students gain skill.

Supports can return when learners struggle.

## 6. Keep adaptivity transparent but not stigmatizing

Do not show badges like “Easy Mode” or “Needs Help.”

Use positive framing:

- Training Assist Available
- Slope Scanner Ready
- Practice Route Unlocked
- Challenge Route Unlocked

## 7. Reward learning behavior

Badges, XP, and stars should reward accuracy, strategy, improvement, and reflection.

Do not reward only speed or grinding.

## 8. Multiplayer is optional application, not the learning backbone

The main campaign teaches. Multiplayer lets students apply skills.

Do not require weaker students to defeat stronger students in order to progress.

## 9. Reduce cognitive overload

The graph must remain readable. Pixel art, music, effects, and UI should support the math rather than distract from it.

## 10. Teacher controls matter

Teachers should be able to tune difficulty and supports for their classroom.
